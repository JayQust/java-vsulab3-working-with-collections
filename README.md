# Сравнение производительности коллекций Java (ArrayList vs LinkedList)

## Цель работы

Данный проект предназначен для сравнения производительности двух основных реализаций интерфейса `List` в Java: **ArrayList** и **LinkedList**.

Тестирование проводится для различных ключевых операций (`add`, `get`, `remove`, `contains`, `add(index, element)`) и при трех разных уровнях нагрузки (малый, средний, большой), чтобы выявить различия в их асимптотической сложности (Big O Notation).

## Структура проекта

Проект реализован как стандартный Maven-проект с использованием Java 17+.
1.  **`pom.xml`**: Файл конфигурации Maven.
2.  **`src/main/java/com/example/lab3/`**:
    * `Main.java`: Точка входа. Определяет уровни нагрузки, запускает все тесты и выводит форматированную таблицу результатов.
    * `SimpleTester.java`: Основной класс-тестер. Содержит логику замера времени для каждой операции (`add`, `get`, `remove`, `contains`, `insertMiddle`).
3.  **`src/test/java/com/example/lab3/`**:
    * `SimpleTesterTest.java`: Юнит-тесты на основе JUnit 5 для проверки корректности работы методов тестера (заполнение списка, удаление, корректность размеров).

## Сборка и запуск

### Требования

* Java Development Kit (JDK) версии 17 или выше.
* Apache Maven.

### 1. Сборка проекта
Выполните команду в корневой папке проекта (`lab3/`):
mvn clean install

### 2. Запуск Unit Тестов
Для проверки корректности логики тестера:
Bashmvn test
### 3. Запуск основного сравнения
Для запуска тестов производительности и вывода итоговой таблицы:
mvn exec:java -Dexec.mainClass="com.example.lab3.Main"


## Ожидаемые результаты
Результаты будут представлены в таблице, показывающей время выполнения операции в миллисекундах (мс).

| Операция      | ArrayList       | LinkedList      | Причина                                   |
|---------------|-----------------|-----------------|-------------------------------------------|
| get(index)    | Быстро (O(1))   | Медленно (O(N)) | ArrayList обращается напрямую по индексу. |
| remove(0)     | Медленно (O(N)) | Быстро (O(1))   | ArrayList должен сдвинуть весь массив.    |
| add(index, E) | Медленно (O(N)) | Быстро (O(1))   | LinkedList просто меняет ссылки.          |
| contains(E)   | Медленно (O(N)) | Медленно (O(N)) | Линейный поиск в обеих коллекциях.        |

Особое внимание следует обратить на то, как сильно увеличивается время выполнения операций O(N) (remove(0), get(index) для LinkedList) при переходе от Малого N к Большому N по сравнению с операциями O(1).
